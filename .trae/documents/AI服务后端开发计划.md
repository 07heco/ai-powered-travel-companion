# AI服务后端开发计划

## 1. 分支管理
- 创建新分支 `AI-service-dev` 用于AI服务的开发
- 确保分支与主分支保持同步

## 2. 核心功能开发

### 2.1 完善AI规划助手
- 实现 `AIServiceImpl` 类，包含AI旅行计划生成逻辑
- 集成OpenAI API或其他开源AI模型
- 实现智能行程推荐算法
- 支持个性化偏好设置
- 提供详细的每日行程生成

### 2.2 实现AR实景导览
- 新增 `ARController` 类，提供AR相关接口
- 实现景点识别API
- 集成图像识别模型（可使用HuggingFace或ModelScope）
- 提供景点详细信息查询
- 实现AR导航数据接口

### 2.3 开发智能预算功能
- 新增 `BudgetController` 类，提供预算管理接口
- 实现预算分析算法
- 支持消费分类和统计
- 提供AI智能建议
- 实现预算预测功能

### 2.4 实现旅伴匹配系统
- 新增 `TravelMateController` 类，提供旅伴匹配接口
- 实现AI匹配算法
- 支持用户兴趣和旅行风格分析
- 提供旅伴推荐功能
- 实现招募帖管理

## 3. 数据模型设计

### 3.1 核心数据表
- `ai_plan` - AI旅行计划表
- `travel_mate` - 旅游伙伴表
- `travel_mate_chat` - 聊天记录表
- `ar_landmark` - AR景点信息表
- `budget` - 预算表
- `expense` - 支出记录表
- `travel_mate_match` - 旅伴匹配记录表
- `recruit_post` - 招募帖表

### 3.2 数据访问层
- 完善所有DAO接口
- 实现对应的Mapper XML文件
- 确保数据库操作的正确性和性能
