# 添加项目根目录缺失文件

## 项目分析

通过分析项目结构，我发现以下问题：

1. **根目录缺少 .gitignore 文件**：没有定义 Git 应该忽略的文件和目录
2. **根目录缺少 Maven wrapper 文件**：缺少 mvnw 和 mvnw.cmd 文件，这些文件用于确保使用正确版本的 Maven
3. **根目录缺少父级 pom.xml 文件**：虽然 backend 目录下有 pom.xml，但根目录可能需要一个总的 pom.xml 来管理整个项目

## 解决方案

### 1. 创建 .gitignore 文件

在项目根目录创建 .gitignore 文件，包含常见的忽略项，如：
- IDE 配置文件
- 构建输出目录
- 依赖目录
- 日志文件
- 环境变量文件

### 2. 添加 Maven wrapper 文件

在项目根目录添加以下文件：
- mvnw：Unix/Mac 系统的 Maven wrapper 脚本
- mvnw.cmd：Windows 系统的 Maven wrapper 脚本
- .mvn/wrapper/maven-wrapper.jar：Maven wrapper 核心文件
- .mvn/wrapper/maven-wrapper.properties：Maven wrapper 配置文件

### 3. 创建根目录 pom.xml 文件

在项目根目录创建一个父级 pom.xml 文件，用于管理整个项目的依赖和构建配置。

## 预期结果

完成这些操作后，项目将具有完整的开发环境配置，便于团队协作和版本控制。
